# ViT 深度估计轻量级解码器方案总结

本文档基于讨论内容，总结了在使用 ViT 提取特征进行密集深度估计时，为了解决常见的伪影（块状边缘、棋盘格）同时保持轻量级结构（替代计算密集的 DPT 架构）的几种具体实现方案。

## 核心挑战与现象
1. **块状/锯齿状边缘**：仅依赖普通 ViT 最后一层的 token（1/16 分辨率），会导致网络缺乏细粒度的空间边界信息。
2. **棋盘格伪影**：使用大倍率的 PixelShuffle（或转置卷积）进行上采样时，由于缺乏相邻像素间的空间混合，会导致平滑区域出现网格状纹理。
3. **内存与计算瓶颈**：若直接在高分辨率下保留多通道特征图，会导致 VRAM 占用和 FLOPs 暴增。

---

## 方案一：全 MLP 多尺度轻量级解码器（强烈推荐）
这是针对普通同构 ViT（如 ViT-Base/Large）最推荐的方案，能够达到类似 DPT 的清晰边界效果，但计算量极低。

### 具体实现步骤：
1. **通道降维（Early Channel Reduction）**：
   - 提取 ViT 的第 4、7、11、23 层的 token（此时各层空间分辨率均为 1/16，通道数为 D，例如 768）。
   - 立即将这 4 个特征分别通过一个轻量级的线性层（MLP 或 1x1 Conv），将通道数大幅降至较小的 $C_{light}$（例如 64 或 128）。
   - *目的：在低分辨率下提前降低通道数，可以极大减少后续高分辨率操作时的计算量和显存占用。*
2. **低成本统一上采样（Cheap Upsampling to Common Resolution）**：
   - 将这 4 个降维并 reshape 成 2D 网格的特征图（此时均为 1/16 分辨率），直接使用无参数的**双线性插值（Bilinear Interpolation）**全部统一上采样到 **1/4** 分辨率。
   - *(注：原讨论中提到先构建 1/4, 1/8, 1/16, 1/32 再统一上采样到 1/4，实际操作中由于起点都是 1/16，直接一步双线性插值缩放到 1/4 即可，避免多此一举。)*
3. **特征拼接与融合（Concatenation & Fusion）**：
   - 在通道维度将这 4 个 1/4 分辨率的特征图拼接（Concatenate）在一起（若 $C_{light}=64$，拼接后总通道数为 256）。
   - 将拼接后的特征通过一个 MLP（1x1 Conv）进行跨尺度特征融合。
4. **上采样与消除伪影（PixelShuffle + DWConv）**：
   - 使用 **PixelShuffle** 将融合后的特征从 1/4 分辨率上采样到最终的 **1/1**（全分辨率）。
   - **关键步骤**：在 PixelShuffle 之后，紧接着添加一个 **3x3 深度可分离卷积（Depthwise Convolution）**。
   - *目的：PixelShuffle 只是进行通道到空间的重排，缺乏空间感知。3x3 DWConv 参数量极小（对单通道特征仅需 9 个参数，FLOPs 极低），但能完美平滑相邻像素，消除棋盘格伪影。*

### 可选优化：半尺寸预测技巧（The "1/2 Scale" Trick）
如果想进一步追求极致的低 FLOPs：
- 将步骤 4 中的 PixelShuffle 目标分辨率设为 **1/2** 或 **1/4** 并在该尺度下预测深度图。
- 在网络的最末端，直接使用无参数的 **双线性插值** 将预测出的深度图拉伸到 **1/1** 全尺寸。因为深度图本身具有局部平滑特性（边缘已由低层特征保证），这种方法效果好且零额外计算成本。

## 方案二：渐进式上采样 + 轻量级空间混合
如果不使用“统一缩放到 1/4 分辨率”的策略，可以采用类似特征金字塔的渐进式方法，逐步放大并混合。

### 具体实现步骤：
1. 提取 ViT 的中间层特征（如 Layer 4, 7, 11, 23）。
2. 从高层语义特征开始，进行逐级放大融合：
   - 对当前特征应用 MLP。
   - 执行小倍率的 **2x PixelShuffle**（例如从 1/16 放大到 1/8）。
   - 紧接着应用一个 **3x3 深度可分离卷积（DWConv）** 来平滑本次放大的边界。
   - 将对应的低一层特征（如 Layer 7，若分辨率不同则上采样对齐到 1/8）与当前特征相加（Add）或拼接（Concat）。
3. 重复“MLP -> 2x PixelShuffle -> 3x3 DWConv -> 融合”的过程，直到达到全分辨率。

---

## 方案三：类 SegFormer 方案（仅限层级式 Backbone）
如果你的骨干网络（Backbone）天然是层级式的（例如 Swin Transformer 或 PvT），它们自身已经输出了 1/4、1/8、1/16、1/32 的特征图。

### 具体实现步骤：
1. 提取层级式网络输出的 4 个不同尺度的特征图。
2. 使用 MLPs（1x1 Conv）将它们分别投影到相同的通道维度。
3. 不使用 PixelShuffle，而是全部使用 **双线性插值（Bilinear Interpolation）** 将它们上采样到 **1/4** 分辨率。
4. 将 4 个特征图拼接在一起。
5. 应用最终的 MLP 融合它们，并预测深度图。
6. 最后使用双线性插值将深度图上采样到全分辨率。
*(注：插值天然比单次大倍率 PixelShuffle 更平滑)*

---

## 方案四：单步大倍率 PixelShuffle 的保底补救方案
如果受限于某些严格的架构要求，**必须**使用普通的 ViT 且只能在最后进行一次单步的 PixelShuffle 放大。

### 具体实现步骤：
1. 提取 Layer 4, 7, 11, 23 的 token（全部为 1/16 分辨率）。
2. 直接在通道维度拼接它们，并应用 MLP 进行处理。
3. 应用单次大倍率的 PixelShuffle（如 16x）直接达到全分辨率。
4. **后处理平滑**：在最后添加 **两层 3x3 标准卷积（带 ReLU）**。
   - *目的：这充当了抗混叠/平滑滤波器，强制混合 PixelShuffle 产生的生硬块状边界。*
   - *缺点：在全分辨率下使用标准 3x3 卷积会显著增加 FLOPs，削弱“轻量级”的优势声明。*
